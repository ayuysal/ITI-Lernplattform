Kapitel 1

Informatik & Algorithmen
Informatik

Informatik ist die Wissenschaft, die sich mit den theoretischen Grundlagen, den Mitteln und Methoden sowie mit der Anwendung der elektronischen Datenverarbeitung beschäftigt, also mit der Informationsverarbeitung unter Einsatz von Computern.

EVA-Prinzip (Eingabe – Verarbeitung – Ausgabe)

Die Datenverarbeitung verläuft nach dem EVA-Prinzip: Zuerst werden Daten eingegeben (Input), dann nach bestimmten Regeln verarbeitet (Process), und schließlich werden die Ergebnisse ausgegeben (Output). Englisch: IPO-Prinzip (Input – Process – Output).

DatenspeicherungLangfristige oder kurzfristige Ablage von DatenArchivierung, Datenbanken
KommunikationVerbindung und Datenaustausch mit anderen RechnernInternet, Netzwerke
SteuerungKontrolle und Regelung externer GeräteWaschmaschine, Autopilot, Roboter
Vorteile der automatisierten Datenverarbeitung

Im Vergleich zur manuellen Datenverarbeitung bietet die automatisierte Verarbeitung: höhere Geschwindigkeit (v.a. bei Routineaufgaben), höhere Zuverlässigkeit (keine Ermüdung, keine Flüchtigkeitsfehler) und geringere Kosten (auf mittlere Sicht).

Algorithmus

Ein Algorithmus ist eine endliche Menge von eindeutig festgelegten Verfahrensvorschriften zur Lösung eines Problems durch eine in der Regel endliche Menge von Einzelschritten. Der Begriff leitet sich vom arabischen Mathematiker al-Hwarizmi (9. Jh.) ab.

VOM PROBLEM ZUM PROGRAMM – 3 SCHRITTE
  1. Problemdefinition – Problem so genau wie möglich beschreiben
  2. Programmentwurf – Algorithmus zur Lösung des Problems entwerfen
  3. Programmierung – Algorithmus in einer Programmiersprache umsetzen

Strukturierte Programmierung

Ein Programm entspricht den Regeln der strukturierten Programmierung, wenn es keinerlei Sprunganweisungen enthält und die Steuerung des Programmablaufs mithilfe der drei Kontrollstrukturen Sequenz, Auswahl und Wiederholung erfolgt. (Grundlage: Dijkstra, späte 1960er-Jahre)

Kapitel 2

Umsetzung von Algorithmen
Kontrollstrukturen

Es gibt drei grundlegende Kontrollstrukturen in der strukturierten Programmierung: Sequenz (Anweisungen nacheinander), Auswahl/Selektion (Verzweigung je nach Bedingung) und Wiederholung/Iteration (Schleife mit Bedingung).

DIE DREI KONTROLLSTRUKTUREN
KontrollstrukturPrinzipStruktogrammC-Syntax
SequenzAnweisungen werden nacheinander ausgeführt, jede genau einmalRechteckige Kästen untereinanderAnweisung1; Anweisung2;
Auswahl (Selektion)Bedingung prüfen → wahr/falsch-ZweigDreieck mit wahr/falsch-Zweigif (Bedingung) { ... } else { ... }
Wiederholung (Iteration)Anweisungsblock wiederholt ausführen, solange Bedingung giltKasten mit Durchführungsbedingung am Fußdo { ... } while(Bedingung);
Struktogramme (Nassi-Shneiderman-Diagramme)

Seit 1973 als Methode zur Strukturierung von Algorithmen etabliert. Jeder Verarbeitungsschritt wird in einem rechteckigen Kasten dargestellt. Vorteile: Sprachunabhängig, leicht lesbar, erzwingen strukturierte Programmierung (keine Sprünge möglich).

do-Schleife (nicht abweisende/fußgesteuerte Schleife)

Die Anweisungen im Schleifenrumpf werden mindestens einmal ausgeführt. Erst am Ende (am „Fuß") wird die Durchführungsbedingung geprüft. Schema in C: do { Anweisungen } while(Bedingung);

Objektorientierte Programmierung (OOP)

Bei der OOP wird jede Aktivität einem Objekt zugeordnet. Objekte gleicher Art werden in Klassen zusammengefasst. Eine Klasse definiert Attribute (Daten/Eigenschaften) und Methoden (Operationen). Die Zusammenfassung von Daten und Operationen in einer Einheit heißt Kapselung.

Zeitkomplexität & Landau-Symbole (Big-O)

Die Zeitkomplexität T(n) beschreibt, wie viele Operationen ein Algorithmus in Abhängigkeit von der Eingabegröße n benötigt. Ist T(n) ≤ c · g(n) für eine Konstante c und alle hinreichend großen n, schreibt man T(n) = O(g(n)).

T(n) = O(g(n)) ⇔ ∃ c > 0 : T(n) ≤ c · g(n) für alle großen n

Kapitel 3

Rechner
Analytische Maschine (Charles Babbage, 19. Jh.)

Obwohl sie nie gebaut wurde, war die Analytische Maschine ein Vorläufer moderner Computer. Ihr Plan sah vor: ein Rechenwerk (Zahnräder, dezimal), einen Zahlenspeicher (1000 Zahlen à 50 Stellen), Ein-/Ausgabegeräte (Lochkarten) und ein Steuerwerk zur Programmsteuerung – inklusive Verzweigungen.

Turing-Maschine (Alan Turing, 1936)

Ein theoretisches Modell für einen programmierbaren Computer. Bestandteile: ein beliebig langes Band mit Speicherzellen, ein Lese-/Schreibkopf (liest, schreibt, bewegt sich L/R), ein endliches Alphabet (inkl. Blank □) und endlich viele Zustände. Programmzeilen haben die Form: (Zustand, gelesenes Zeichen) → (neuer Zustand, zu schreibendes Zeichen, Bewegung).

Bedeutung der Turing-Maschine

Alles, was überhaupt berechenbar ist, kann mithilfe einer Turing-Maschine berechnet werden. Eine universelle Turing-Maschine kann jede spezielle Turing-Maschine simulieren – das einfachste Modell eines Universalcomputers.

Von-Neumann-Architektur (John von Neumann, 1940er)

Grundprinzip: Gespeichertes Programm – Befehle und Daten liegen im gleichen Arbeitsspeicher. Befehle werden sequenziell abgearbeitet. Die Architektur bildet die Grundlage der meisten heutigen Computer.

Dualzahlen
Stellenwertsystem

In einem Stellenwertsystem bestimmt die Position einer Ziffer ihren Wert. Jede Stelle wird mit der entsprechenden Potenz der Basis multipliziert. Dezimalsystem: Basis 10; Dualsystem: Basis 2.

Dual → Dezimal Umrechnung

Jede Dualziffer wird mit der zugehörigen Zweierpotenz multipliziert und die Ergebnisse werden addiert:

an·2n + an-1·2n-1 + … + a1·2 + a0

Beispiel: 10111₂ = 1·16 + 0·8 + 1·4 + 1·2 + 1·1 = 23

Dezimal → Dual Umrechnung (Divisionsmethode)

Verfahren: Die Dezimalzahl fortlaufend durch 2 teilen, jeweils den Rest notieren. Abbruch wenn Ergebnis = 0. Reste von unten nach oben gelesen ergeben die Dualzahl.

Beispiel: 13 : 2 = 6 R1 → 6 : 2 = 3 R0 → 3 : 2 = 1 R1 → 1 : 2 = 0 R1 → Ergebnis: 1101₂

Man addiert dual genauso wie dezimal, nur eben zur Basis 2. Übertrag entsteht, wenn die Basiszahl (2) erreicht wird.

Einserkomplement & Zweierkomplement

Einserkomplement: Alle Nullen durch Einsen ersetzen und umgekehrt. Zweierkomplement: Einserkomplement + 1₂ addieren. Damit kann man Subtraktion auf Addition zurückführen: a − b = a + Zweierkomplement(b), danach die führende 1 streichen.

Beispiel: 11001₂ − 01010₂ mit Zweierkomplement

Einserkomplement von 01010: 10101 → +1 = Zweierkomplement: 10110
11001 + 10110 = 101111 → führende 1 streichen → Ergebnis: 01111₂ = 15

HEXADEZIMAL ↔ DUAL UMRECHNUNG

Dual → Hex: Von rechts nach links in Vierergruppen teilen, jede Gruppe in Hex-Ziffer umwandeln.

Hex → Dual: Jede Hex-Ziffer in 4-Bit-Dualzahl auflösen.

Oktal: Gleiches Prinzip, aber mit Dreiergruppen (Ziffern 0–7).

ASCII-Code & Unicode

ASCII (American Standard Code for Information Interchange): Ursprünglich 7 Bit = 128 Zeichen, erweitert auf 8 Bit = 256 Zeichen. Buchstaben A–Z = 41₁₆–5A₁₆, a–z = 61₁₆–7A₁₆.
Unicode: Verwendet mehrere Bytes (2 Byte = 65.536 Zeichen, 4 Byte = über 4 Mrd.) für alle Schriftsysteme weltweit.

Addierwerk (mehrstellige Addition)

Ein Addierwerk für n-stellige Dualzahlen besteht aus 1 Halbaddierer (für die niedrigste Stelle) und (n−1) Volladdierern (für alle weiteren Stellen, die jeweils den Übertrag der vorherigen Stelle mitverarbeiten). Der letzte Übertrag ist die (n+1)-te Stelle des Ergebnisses.
