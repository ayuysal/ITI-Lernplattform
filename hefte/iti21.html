Kapitel 1–2

Einführung & Internet – das Netz der Netze
Definition

Informations- & Kommunikationstechnologien (IuK/IKT): Technologien zur Verarbeitung, Speicherung und Übertragung von Informationen in elektronischer Form.

Definition

Internet (FNC 1995): Globales Informationssystem, das auf einem weltweit einheitlichen IP-Adressraum basiert, die TCP/IP-Protokollfamilie nutzt und öffentliche und private Dienste bereitstellt.

Wichtig: Netz ≠ Netzwerk

„Netzwerk" ist ein Anglizismus (Übersetzungsfehler von „network"). Korrekt: Computernetz. In den Studienheften wird „Netz" verwendet. Ausnahme: wenn sich „Netzwerk" in einem Kontext fest etabliert hat (z. B. Netzwerkkabel).

2.1 Warum heißt das Internet „Netz der Netze"? → Es ist ein dezentrales globales Netz, das aus einer Vielzahl miteinander verbundener Teilnetze besteht.


2.2 Wie ist das Internet organisiert? → Dachorganisation ISOC; IETF entwickelt Standards (RFCs); IANA/ICANN vergibt Adressen und Domainnamen; RIPE NCC für Europa.


2.3 Host = Gerät am Rand des Netzes. Dienst = technische Funktionskomponente (z. B. WWW). Protokoll = Regelwerk für Kommunikation (z. B. TCP/IP, HTTP).

Kapitel 3

Einordnung, Ziele & Anforderungen
Definition 3.1 – Computernetz

Ein Computernetz ist ein Verbund von zwei oder mehr autonomen Computern und anderen Systemen (Hosts), der die Kommunikation und den gemeinsamen Zugriff auf Ressourcen und Dienste ermöglicht.

Definition 3.2 – Verteiltes System

Eine Sammlung unabhängiger Computer, die dem Benutzer wie ein einziges kohärentes System erscheint (Tanenbaum). Basiert auf einem Computernetz + zusätzliche Software.

Abgrenzung: Computernetz ↔ Verteiltes System ↔ Multiprozessorsystem

Computernetz: Autonome Computer, größere Latenz.
Verteiltes System: Erscheint als ein System (z. B. WWW).
Multiprozessorsystem: Enge Kopplung, gemeinsamer Speicher, sehr geringe Latenz.

Definition 3.4 – Netzdienst

Ein Netzdienst stellt Anwendern bzw. Geräten eine in sich geschlossene Funktionskomponente zur Kommunikation oder gemeinsamen Nutzung von Ressourcen zur Verfügung. Technisch realisiert über Protokolle. Beispiel: WWW → HTTP/HTTPS.

Vermittlungsprinzipien (PRÜFUNGSRELEVANT)
Leitungsvermittlung (Circuit Switching)

Dedizierte physikalische Verbindung für die gesamte Sitzung. Ressourcen exklusiv reserviert.
✅ Garantierte Bandbreite, geringe Latenz
❌ Schlechte Leitungsauslastung (z. B. Sprachpausen)
Beispiel: Traditionelles Telefonnetz

Paketvermittlung (Packet Switching)

Nachricht wird in Pakete zerlegt, jedes Paket einzeln vermittelt. Keine Reservierung.
✅ Bessere Leitungsauslastung, Fehlertoleranz
❌ Keine garantierte Bandbreite, Reihenfolge-Overhead
Beispiel: Internet (TCP/IP)

Kapitel 5

Kenngrößen in Netzen
⚠ Wichtig bei Berechnungen

Bei Übertragungsraten: Basis 10 (nicht 2!). Also: 1 kbit/s = 1000 bit/s.
Bei Speicher/Datenmengen: Basis 2. 1 KiB = 1024 Byte. 1 MiB = 1 048 576 Byte. Einheit immer prüfen!

Formeln (alle auswendig lernen)

Übertragungsrate:

b = D / t   |   t = D / b

Übertragungsverzögerung:

t_Ü = D / b     (Datenmenge / Übertragungsrate)

Signalausbreitungszeit:

t_a = d / v     (Entfernung / Ausbreitungsgeschwindigkeit)

Gesamtlatenz (vereinfacht):

t_L = t_Ü + t_a     (+ Verarbeitungs- & Wartezeit)

Kanalvolumen:

V = Latenz × Übertragungsrate     [in bit]

Bitzeit:

t_bit = 1 / b

Latenz-Komponenten (paketvermittelte Netze)
Processing Delay (t_v)

Verarbeitungszeit im Knoten: Entscheidung, auf welche Ausgangsleitung die Daten kommen

Queuing Delay (t_w)

Wartezeit in der Warteschlange der ausgehenden Leitung

Transmission Delay (t_Ü)

Zeit, bis das letzte Bit auf den Link gelegt ist: D / b

Propagation Delay (t_a)

Signalausbreitungszeit durch das Medium: d / v ≈ ⅔ Lichtgeschwindigkeit

Latenz & Sprachqualität (ITU-T G.114)

≤ 150 ms → gute Sprachqualität | ≤ 400 ms → akzeptabel | > 600 ms → keine Kommunikation möglich.
Paketverlustrate sollte < 5 % betragen.

Serielle Übertragung

ASYNCHRON

  • Synchronisation je Zeichen neu
  • Start- und Stoppbits notwendig → Overhead
  • Langsamer als synchron

SYNCHRON (in Netzen Standard)

  • Synchronisation während gesamter Übertragung
  • Sendetakt wird mit Daten übertragen
  • Frame mit Flag begrenzt (HDLC: 01111110)
  • Bit Stuffing nach 5 aufeinanderfolgenden 1-Bits

Kapitel 6

Schichtenmodelle & Protokolle – OSI & TCP/IP
Warum Schichtenmodelle?

Komplexe Kommunikationsaufgaben werden in hierarchisch geordnete Teilaufgaben zerlegt. Jede Schicht erbringt Dienste für die darüber liegende und nutzt Dienste der darunterliegenden. → Modularität, Interoperabilität, einfachere Entwicklung.

Kapselung & Entkapselung
Kapselung (Sender, von oben nach unten)

Jede Schicht fügt dem Datenpaket einen eigenen Header hinzu (ggf. auch Trailer). Die Anwendungsdaten werden sukzessive in Protokolleinheiten der tieferen Schichten verpackt.

Daten → Segment (TCP) → Paket (IP) → Frame (Ethernet) → Bits

Entkapselung (Empfänger, von unten nach oben)

Jede Schicht liest und entfernt ihren Header und reicht die Daten an die nächsthöhere Schicht weiter.
