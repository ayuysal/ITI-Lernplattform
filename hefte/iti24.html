Kapitel 1

Einführung & Überblick
TCP/IP-Protokollfamilie

Sammlung von über 500 Protokollen, die die Kommunikation im Internet regeln. Benannt nach den beiden wichtigsten Protokollen: TCP (Transmission Control Protocol) und IP (Internet Protocol).

TCP/IP-Referenzmodell (4 Schichten)
SchichtNameProtokolle (Beispiele)PDU
4AnwendungsschichtHTTP, FTP, SMTP, DNS, DHCPDaten
3TransportschichtTCP, UDPSegment / Datagramm
2Netzwerkschicht (Internet)IP, ICMP, ARP, IGMPPaket
1NetzzugangsschichtEthernet, WLAN, PPPFrame / Bits
Netzwerkschicht – Kernaufgaben
  • Logische Adressierung – IP-Adressen zur Identifikation von Hosts weltweit
  • Routing – Optimale Wegberechnung durch das Netz
  • Forwarding – Weiterleitung pro Paket anhand der Forwardingtabelle
  • Best-Effort-Dienst – Keine Garantie für Zustellung, Reihenfolge oder Fehlerfreiheit
Leitungsvermittlung vs. Paketvermittlung

Leitungsvermittlung: Fester Weg für gesamte Verbindung (Telefon). Alle Daten nehmen denselben Weg.
Paketvermittlung: Jedes Paket wird einzeln geroutet. Pakete können verschiedene Wege nehmen. Bessere Netzauslastung, aber mehr Overhead.

Transportschicht – Kernaufgaben
  • Ende-zu-Ende-Kommunikation – Zwischen Anwendungsprozessen auf den Hosts
  • Multiplexing/Demultiplexing – Zuordnung über Portnummern
  • TCP: verbindungsorientiert, zuverlässig, Flusskontrolle, Staukontrolle
  • UDP: verbindungslos, unzuverlässig, minimal Overhead
  • Router werten nur bis Schicht 3 aus – Transportprotokolle laufen auf den Hosts
RFC (Request for Comments)

Offene Standards der IETF (Internet Engineering Task Force). Jedes Internetprotokoll wird in einem RFC spezifiziert. RFCs werden fortlaufend nummeriert und können durch neuere ersetzt werden.

Kapitel 2

IP-Adressierung
IPv4-Adresse

32 Bit = 4 Byte, dezimale Punkt-Notation (z.B. 192.168.1.1). Maximal 2³² ≈ 4,3 Milliarden Adressen. Besteht aus Netzanteil (Netz-ID) und Hostanteil (Host-ID).

Private Adressen werden im Internet nicht geroutet. Für Internetzugang ist NAT erforderlich.

CIDR (Classless Interdomain Routing)

Flexible Aufteilung statt starrer Klassen. Notation: IP/Präfixlänge (z.B. 192.168.1.0/24).

Adressierbare Hosts = 2(32−Präfixlänge) − 2

Minus 2: eine Adresse für das Netz (alle Hostbits = 0), eine für Broadcast (alle Hostbits = 1).

IPv6-Adresse

128 Bit = 16 Byte, hexadezimale Notation mit Doppelpunkten (z.B. 2001:0db8:85a3::8a2e:0370:7334). Adressraum: 2¹²⁸ ≈ 3,4 × 10³⁸ Adressen.

Unique LocalFC00::/7Privat (wie RFC 1918 bei IPv4)
MulticastFF00::/8Gruppe von Hosts
Loopback::1/128Localhost
Interface Identifier & SLAAC
  • Interface ID – 64 Bit, gebildet aus MAC-Adresse (EUI-64): OUI + FFFE + CID, U/L-Bit invertiert
  • SLAAC (Stateless Address Autoconfiguration) – Host bildet Adresse selbst: Präfix vom Router (Router Advertisement) + Interface ID
  • Privacy Extensions – Zufällige Interface ID statt MAC-basiert (Datenschutz)
  • DAD (Duplicate Address Detection) – Prüfung auf Adressduplikate via Neighbor Solicitation
Spezielle IPv4-Adressen
  • 127.0.0.1 – Loopback (localhost)
  • 0.0.0.0 – Unbekannte/Standardroute
  • 255.255.255.255 – Limited Broadcast
  • 169.254.x.x – APIPA (Automatic Private IP Addressing, bei DHCP-Ausfall)

Fragmentierung

Wenn ein IP-Paket die MTU (Maximum Transmission Unit, Ethernet: 1500 Byte) überschreitet, wird es fragmentiert. Jedes Fragment enthält eigenen IP-Header (20 Byte).

Nutzdaten pro Fragment = MTU − IP-Header = 1500 − 20 = 1480 Byte

Zusammensetzung erst beim Empfänger. DF-Flag verhindert Fragmentierung → bei Überschreitung: ICMP „Fragmentation Needed".

ICMP (Internet Control Message Protocol)

Hilfsprotokoll für Fehlermeldungen und Diagnosefunktionen in IP-Netzen. Wird von Ping und Traceroute genutzt.

Wichtige ICMP-Typen
TypNameBedeutung
0Echo ReplyAntwort auf Ping
3Destination UnreachableZiel nicht erreichbar (diverse Codes)
8Echo RequestPing-Anfrage
11Time ExceededTTL abgelaufen (Basis für Traceroute)

Traceroute: Sendet Pakete mit aufsteigender TTL (1, 2, 3, …). Jeder Router auf dem Weg sendet „TTL Exceeded" zurück → Route wird sichtbar.

ARP (Address Resolution Protocol)
  • Aufgabe: Auflösung von IP-Adresse → MAC-Adresse im lokalen Netz
  • ARP-Request: Broadcast (FF:FF:FF:FF:FF:FF) – „Wer hat IP x.x.x.x?"
  • ARP-Reply: Unicast – „Ich habe x.x.x.x, meine MAC ist …"
  • ARP-Cache: Zwischenspeicher für aufgelöste Zuordnungen (arp -a / arp -d)
  • Proxy ARP: Router antwortet für Hosts in einem anderen Netz
  • RARP: Umgekehrt: MAC → IP (historisch, durch DHCP abgelöst)
NDP (Neighbor Discovery Protocol) – IPv6
  • Ersetzt ARP, ICMP Router Discovery und DHCP-Teile bei IPv6
  • Neighbor Solicitation – Multicast statt Broadcast (effizienter als ARP)
  • Neighbor Advertisement – Antwort mit Link-Layer-Adresse
  • Router Solicitation / Advertisement – Host fragt nach Routern, Router antworten mit Präfix
  • DAD – Duplicate Address Detection über Neighbor Solicitation
  • NUD – Neighbor Unreachability Detection (Erreichbarkeitsprüfung)
DNS (Domain Name System)
  • Aufgabe: Auflösung von Domainnamen in IP-Adressen (und umgekehrt)
  • Hierarchie: Root-Server → TLD-Server (.de, .com) → Authoritative Server
  • Rekursive Abfrage: DNS-Resolver erledigt gesamte Auflösung für den Client
  • Iterative Abfrage: Jeder Server verweist auf den nächsten zuständigen Server
  • DNS-Cache: Zwischenspeicher für schnellere Auflösung (TTL begrenzt)
  • Record-Typen: A (IPv4), AAAA (IPv6), CNAME (Alias), MX (Mail), NS (Nameserver)
  • Transport über UDP Port 53 (Standardanfragen) oder TCP Port 53 (Zonentransfer)
DHCP (Dynamic Host Configuration Protocol)
  • Aufgabe: Automatische Vergabe von IP-Konfiguration (IP, Maske, Gateway, DNS)
  • DORA-Prozess: Discover → Offer → Request → Acknowledge
  • Client sendet Broadcast (DHCP Discover), Server antwortet mit Angebot
  • Lease Time: Gültigkeitsdauer der zugewiesenen Adresse
  • DHCPv6: Für IPv6, ergänzt SLAAC um zusätzliche Konfigurationsparameter
NAT (Network Address Translation)
  • Aufgabe: Übersetzung privater in öffentliche IP-Adressen am Router
  • Static NAT: 1:1-Zuordnung (private → öffentliche Adresse)
  • Dynamic NAT: Pool öffentlicher Adressen, dynamische Zuordnung
  • PAT / NAT Overload: Viele private Adressen → eine öffentliche Adresse (unterschieden durch Portnummern)
  • Verhindert direkte Erreichbarkeit interner Hosts von außen
  • NAT-Tabelle speichert Zuordnungen für aktive Verbindungen

Kapitel 4

Routing-Verfahren & Routing-Algorithmen
Routing vs. Forwarding

Routing: Berechnung des optimalen Weges durch das Netz mittels Routing-Algorithmen → Ergebnis: Routingtabelle.
Forwarding: Weiterleitung eines konkreten Pakets anhand der Forwardingtabelle (Zielnetz → Next Hop + Ausgangs-Interface).

Forwardingtabelle – Aufbau
ZielnetzNetzmaskeNext HopInterfaceMetrik
192.168.1.0/24direkt verbundeneth00
10.0.0.0/8192.168.1.1eth01
0.0.0.0/0192.168.1.254eth0–

Longest Prefix Match: Router wählt den Eintrag mit der längsten passenden Netzmaske. Default-Route (0.0.0.0/0) wird als letztes verwendet.

Distanz-Vektor-Algorithmus (Bellman-Ford)

Jeder Router kennt nur seine direkten Nachbarn und deren Kosten. Router tauschen regelmäßig ihre Distanzvektoren aus. Iterative Annäherung an optimale Routen.

DV-Algorithmus – Ablauf
  • Schritt 1: Jeder Router initialisiert Tabelle mit direkten Nachbarn (Kosten bekannt)
  • Schritt 2: Router senden Distanzvektoren an Nachbarn (Ziel + Kosten)
  • Schritt 3: Bei Empfang: Prüfen ob neuer Weg kürzer → Tabelle aktualisieren
  • Schritt 4: Wiederholen bis Konvergenz (keine Änderungen mehr)
  • Formel: d(x,y) = min { c(x,v) + d(v,y) } für alle Nachbarn v
Count-to-Infinity-Problem

Bei Ausfall einer Verbindung bietet ein Nachbar eine veraltete, scheinbar günstigere Route an → Routingschleife entsteht. Kosten zählen bis „unendlich" hoch.

  • Split Horizon: Route nicht an den Nachbarn zurücksenden, von dem sie gelernt wurde
  • Poison Reverse: Route mit ∞ Kosten an Herkunftsrouter senden
  • Triggered Updates: Sofortige Benachrichtigung bei Änderung (statt periodisch)
  • Hold-on Timer: Neue Informationen zeitlich sperren
  • Hop Count Limit: Maximum festlegen (z.B. 15 bei RIP → 16 = ∞)
Link-State-Algorithmus (Dijkstra)

Jeder Router kennt die gesamte Netztopologie. Aufbau einer Topologie-Datenbank durch Austausch von LSA-Paketen (Link-State Advertisements). Berechnung des kürzesten Weges mit Dijkstra-Algorithmus.

Hierarchisches Routing & Autonome Systeme
  • Autonomes System (AS): Gruppe von Routern unter einer gemeinsamen Verwaltung und technischen Kontrolle
  • Intra-AS-Routing (IGP): Innerhalb eines AS → RIP, OSPF
  • Inter-AS-Routing (EGP): Zwischen verschiedenen AS → BGP
  • Gateway-Router: Router an der Grenze eines AS, verbindet mit anderen AS
  • Jedes AS hat eine eindeutige AS-Nummer (ASN), vergeben durch RIR (z.B. RIPE NCC)

Kapitel 5

Routing-Protokolle
RIP (Routing Information Protocol)
  • Typ: Intra-AS, Distanz-Vektor-Algorithmus
  • Metrik: Hop Count (Anzahl der Router bis zum Ziel)
  • Maximum: 15 Hops (16 = unerreichbar / ∞)
  • Update: Alle 30 Sekunden komplette Routingtabelle an Nachbarn
  • RIPv1: Klassenbasiert, keine Netzmaske im Update → kein CIDR/VLSM
  • RIPv2: Unterstützt CIDR, VLSM, Authentisierung, Multicast (224.0.0.9)
  • RIPng: RIP für IPv6, UDP Port 521
  • Nachteil: Langsame Konvergenz, Count to Infinity, nur für kleine Netze
OSPF (Open Shortest Path First)
  • Typ: Intra-AS, Link-State-Algorithmus (Dijkstra)
  • Metrik: Kosten = Referenzbandbreite / Interface-Bandbreite (Standard: 10⁸ / Bandbreite)
  • LSA-Flooding: Topologieänderungen werden sofort an alle Router weitergegeben
  • Areas: Hierarchische Unterteilung für Skalierbarkeit. Area 0 = Backbone
  • Designated Router (DR): Reduziert LSA-Verkehr in Multi-Access-Netzen
  • Authentisierung: Simple Password oder MD5
  • Konvergenz: Schnell (nur bei Änderungen, Refresh alle ~30 min)
  • Vorteil: Skaliert gut, schnelle Konvergenz, kein Count to Infinity
OSPF-Kostenberechnung

Kosten = Referenzbandbreite / Interface-Bandbreite

Standard-Referenz: 10⁸ bit/s. Beispiel: 100 Mbit/s-Link → Kosten = 10⁸/10⁸ = 1. 10 Mbit/s-Link → Kosten = 10⁸/10⁷ = 10.

BGP (Border Gateway Protocol)
  • Typ: Inter-AS, Pfadvektor-Algorithmus
  • Aufgabe: Routing zwischen Autonomen Systemen (das Routing-Protokoll des Internets)
  • Metrik: AS-Pfad (Liste der durchquerten AS) + weitere Attribute (z.B. Local Preference)
  • eBGP: Zwischen verschiedenen AS (External BGP)
  • iBGP: Innerhalb eines AS (Internal BGP), verteilt extern gelernte Routen
  • TCP Port 179: BGP nutzt TCP für zuverlässige Nachbarschaftsbeziehungen
  • Policy-based: Administrator kann Routing-Entscheidungen durch Policies beeinflussen
  • Routenaggregation: Zusammenfassung von Routen (Supernetting) reduziert Tabelleneinträge

IPv6-Routing
  • RIPng: RIP für IPv6, UDP Port 521, Multicast FF02::9
  • OSPFv3: OSPF für IPv6, Link-lokale Adressen für Nachbarschaft
  • MP-BGP: Multiprotocol BGP, unterstützt IPv4 und IPv6 gleichzeitig
  • Routing-Prinzipien bleiben gleich, Adressformat ändert sich

Kapitel 6

Transportprotokolle
Transportschicht

Ende-zu-Ende-Kommunikation zwischen Anwendungsprozessen. Identifikation der Prozesse über Portnummern. Zwei Protokolle: TCP (zuverlässig) und UDP (schnell).

Portnummern
BereichNameBeschreibung
0 – 1023Well-known PortsReserviert für Standarddienste (HTTP=80, HTTPS=443, FTP=21, SSH=22, DNS=53, SMTP=25)
1024 – 49151Registered PortsRegistriert bei IANA für bestimmte Anwendungen
49152 – 65535Dynamic/Private PortsDynamisch vom Betriebssystem vergeben (Quellports)

Socket: IP-Adresse + Portnummer (z.B. 192.168.1.1:80). Ein Socket-Paar definiert eine Verbindung eindeutig.

TCP (Transmission Control Protocol)
  • Verbindungsorientiert – 3-Wege-Handshake zum Verbindungsaufbau
  • Zuverlässig – Sequenznummern, Bestätigungen (ACK), Neuübertragung bei Verlust
  • Reihenfolge – Segmente werden in richtiger Reihenfolge zusammengesetzt
  • Flusskontrolle – Sliding Window, Empfänger gibt Fenstergröße (rwnd) vor
  • Staukontrolle – Slow Start, Congestion Avoidance, Anpassung der Senderate
  • Vollduplex – Gleichzeitiges Senden und Empfangen
TCP – 3-Wege-Handshake

SYN-Flooding (DoS-Angriff)

Angreifer sendet massenweise SYN-Pakete mit gefälschten Quelladressen. Server reserviert Ressourcen (TCB) für jede halboffene Verbindung → Speicher und Verbindungslimit erschöpft → Server nicht mehr erreichbar. Gegenmaßnahme: SYN-Cookies.

UDP (User Datagram Protocol)
  • Verbindungslos – Kein Handshake, kein Verbindungsaufbau
  • Unzuverlässig – Keine Bestätigungen, keine Neuübertragung
  • Minimaler Overhead – Header nur 8 Byte (Quellport, Zielport, Länge, Prüfsumme)
  • Schnell – Ideal für zeitkritische Anwendungen
  • Einsatz: DNS-Anfragen, VoIP, Video-Streaming, Onlinespiele, DHCP, SNMP

ping – Erreichbarkeitstest
  • Funktion: Sendet ICMP Echo Request, wartet auf Echo Reply
  • Antwortzeit (RTT): Hin- und Rückweg des Pakets
  • TTL: Wird in Antwort angezeigt → Rückschluss auf Hops
  • ping -t <IP> – Dauerhaft senden (Stopp: STRG+C)
  • ping -l <Größe> <IP> – Pufferlänge ändern
  • ping -w <ms> <IP> – Timeout setzen
Systematische Fehlerdiagnose mit Ping

Schrittweises Testen: 1. ping localhost (127.0.0.1) → 2. ping eigene IP → 3. ping Gateway → 4. ping entfernter Host → 5. ping Domainname (DNS-Test). Fehler grenzt Problem ein.

tracert / traceroute
  • Funktion: Verfolgt den Weg eines Pakets durch das Netz
  • Sendet Pakete mit aufsteigender TTL (1, 2, 3, …)
  • Jeder Router sendet „Time Exceeded" zurück → Route wird sichtbar
  • *** = Router blockiert ICMP (Timeout)
  • tracert <Ziel> (Windows) / traceroute <Ziel> (Linux)
